# 实验原始记录

## 一、Prompt设计

### 1.1 智能问答系统提示词

```
你是一个智能数据分析助手，专门帮助用户分析销售数据。

你的能力包括：
1. 回答关于销售数据的问题
2. 解释数据趋势和模式
3. 提供数据洞察和建议
4. 帮助用户理解统计指标

当用户询问数据相关问题时，请基于提供的数据上下文给出准确、有帮助的回答。
如果需要执行SQL查询来获取数据，请说明你的分析思路。

请用中文回复，保持专业且易于理解。
```

### 1.2 SQL生成提示词

```
请将以下自然语言查询转换为SQL语句。

表结构：
{table_schema}

自然语言查询：{natural_language_query}

请只返回SQL语句，不需要其他解释。
```

### 1.3 数据上下文注入格式

```
当前数据上下文：
数据概览：共500条记录，11列
```

## 二、测试样例与模型输出

### 测试样例1：产品类别销售查询

**输入：**
```
各产品类别的销售额是多少？
```

**Prompt构建：**
```
系统：你是一个智能数据分析助手...

用户：各产品类别的销售额是多少？
```

**模型输出：**
```
根据数据，各产品类别的销售额如下：

1. 电子产品：9,457,015.34元（占比66.3%）
2. 运动器材：2,425,211.72元（占比17.0%）
3. 服装：1,380,130.78元（占比9.7%）
4. 家居用品：715,732.35元（占比5.0%）
5. 食品：297,800.52元（占比2.1%）

可以看出，电子产品是销售额最高的类别，占据了总销售额的近三分之二。建议重点关注电子产品的销售策略和库存管理。
```

## 五、实验总结

### 5.1 成功案例统计

| 类型 | 测试数 | 成功数 | 成功率 |
|------|--------|--------|--------|
| SQL生成 | 10 | 10 | 100% |
| 智能问答 | 10 | 9 | 90% |
| 可视化 | 6 | 6 | 100% |
| 边界测试 | 4 | 4 | 100% |

### 5.2 失败案例分析

**案例1：语义歧义**
- 输入："哪个产品卖得最好？"
- 问题："卖得最好"定义不明确
- 改进：系统可主动询问用户具体指标

**案例2：复杂SQL**
- 输入："找出每个地区销售额排名前3的产品"
- 问题：需要窗口函数，SQL较复杂
- 改进：优化Prompt或增加SQL验证机制
